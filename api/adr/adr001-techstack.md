Requirments:

Backend :

Coding language, API and database chosen should be widely used and supported.
Avaliability of libraries and documentation should be evaluated.
Readability over performance should be preferred.
Developers working on the project should be familiar with the coding language.
We should consider leveraging existing code/experience from other projects.

Conclusion :

We decided to use Go as language for the dapla-api.
We decided to use Postgres for database.
We decided to use the existing Nais API as inspiration/basis for our new dapla-api.
Architectural decisions should be documented here in adr files.
